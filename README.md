Ship-hull signed distance functions (SDFs) for
[WaterLily.jl](https://github.com/WaterLily-jl/WaterLily.jl).
Three flavours of hull are provided:

1. **`Wigley`** — the canonical academic-test parabolic hull. Exact
   analytic SDF.
2. **`Containership`** — parallel-midbody + linear-tapered ends.
   Adjustable block coefficient via `par_frac`. Stand-in for the
   KCS/DTC/DTMB-5415 family until offset data is wired in.
3. **`TabulatedHull`** — trilinear interpolation of a pre-sampled SDF
   grid. The path for real hulls with no closed-form distance: feed in
   a sampled SDF array from a triangle mesh or offset table.

The package is part of an eight-package stack:

| Package | Role |
|---|---|
| WaterLily | Cartesian NS + BDIM substrate |
| Turbulence.jl | LES sub-grid (Smagorinsky, WALE) |
| VoF.jl | MULES free-surface |

| Propellers.jl | Actuator-disk family |
| LiftingSurfaces.jl | VLM-resolved rudder + rotor |
| ShipFlow.jl | Integration scripts |

## Quick start

```julia
using WaterLily, ShipShapes
using StaticArrays: SVector

# Wigley hull: parabolic-section, analytic SDF
L, B, T = 64.0, 14.0, 8.0
hull = Wigley(; L, B, T)

# Translate to the simulation centre via the AutoBody `map`:
hull_xc, hull_yc, hull_zc = 32.0, 32.0, 24.0
hull = Wigley(; L, B, T,
    map = (x, t) -> SVector(x[1] - hull_xc,
                            x[2] - hull_yc,
                            x[3] - hull_zc))

# 6-DOF? Add a deck for metacentric restoring:
hull = Wigley(; L, B, T, deck_h = 4.0)

# Drop into WaterLily:
sim = Simulation((128, 64, 48), (1.0, 0.0, 0.0), L;
    body = hull, perdir = (2,), exitBC = true, U = 1.0)
```

```julia
# Containership-style stand-in (Cb ≈ 0.75 by default)
hull = Containership(; L=64.0, B=14.0, T=8.0,
    par_frac=0.5,   # parallel-midbody fraction → Cb ≈ (1+par_frac)/2
    deck_h=4.0)
```

```julia
# Tabulated hull from a pre-sampled SDF
grid = load_sdf_array("kcs_offsets.h5")   # your data-loading function
hull_sdf = TabulatedHull(grid;
    origin  = SVector(0.0, 0.0, 0.0),
    spacing = SVector(0.1, 0.1, 0.1))
# Wrap as AutoBody:
hull = AutoBody((x, t) -> hull_sdf(x), (x, t) -> x)
```

## Wigley geometry

Wigley's parabolic mathematical hull is defined by

```
y(x, z) / (B/2) = (1 − (2x/L)²) · (1 − (z/T)²)
```

valid on `x ∈ [-L/2, L/2]`, `z ∈ [-T, 0]` (waterline at `z=0`). The
exact analytic SDF in this package handles:

- Interior points (negative distance, equal to negative min of side /
  top / keel exit distance).
- Exterior points (positive distance to clamped surface point).
- Optional deck extension `z ∈ [0, deck_h]`, with the deck having the
  same plan-view shape as the waterline cross-section. Required for
  pitch-stable 6-DOF integration — without it the hull has zero
  metacentric restoring moment.

Displaced volume: `wigley_volume(L, B, T) = 4·L·B·T/9`.

## Containership geometry

The `Containership` SDF approximates the KCS/DTC/DTMB family by
combining:

- A parallel midbody of length `par_frac · L` at full beam `B` and
  full draught `T`.
- Two linear-tapered end-caps reducing the half-beam to zero at the
  bow and stern.
- Vertical sides (no flare or tumblehome).

Block coefficient: `Cb ≈ (1 + par_frac) / 2`. So:

- `par_frac=0.30` → Cb≈0.65 (KCS-like)
- `par_frac=0.50` → Cb≈0.75 (DTC-like, default)
- `par_frac=0.86` → Cb≈0.93 (VLCC-like)

**This is a stand-in, not real KCS.** Real KCS has Cb=0.651, a sharp
bulbous bow, transom stern, and a non-trivial bilge radius — none of
which are captured by the analytic form. The stand-in is useful for
parametric sweeps over Cb (see
[`ShipFlow.jl/RESULTS-cb-vs-Jself.md`](https://github.com/pankgeorg/ShipFlow.jl/blob/main/RESULTS-cb-vs-Jself.md))
but should not be cross-compared against published KCS resistance data.

## TabulatedHull

For real hulls with no analytic SDF, sample the SDF once on a uniform
grid (e.g. via a triangle-mesh distance computation, or by computing
the distance from a marching-cubes output) and pass it as a
`TabulatedHull`. Queries between sample points use trilinear
interpolation. The sampled grid must be wider than the hull on all
sides — keep the body's surface at least one cell away from the
sampled-box boundary.

## Limitations

- **No actual KCS/DTC/DTMB-5415 offsets.** The `Containership` is a
  parametric approximation, not real ship geometry. Validation against
  published towing-tank data for those hulls would need a
  `TabulatedHull` populated from real offset tables.
- **Vertical sides only.** No bow flare, no tumblehome, no bilge
  radius, no transom — all simplifications relative to real hulls.
- **No appendages.** Bulbous bow, rudder, propeller boss, bilge keels,
  fins are all missing. Some are addressed in
  [LiftingSurfaces.jl](https://github.com/pankgeorg/LiftingSurfaces.jl)
  (rudder), others are out of scope.
